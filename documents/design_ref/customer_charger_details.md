# 충전기 관리	
## 충전기 목록 조회	
- provisioning을 통해 설치될 예정인 충전기와 서버간 연동이 정상적으로 성공하면 해당 충전기정보가 이곳에서 조회될 수 있다.
- 정상적으로 등록된 충전기들의 목록을 조회할 수 있다
- 기본으로 20개가 한 화면에 보여진다
- 한화면이 넘는 경우에는 pagenation을 제공하여야 한다
- 한 화면에 보여질 개수를 설정할 수 있고 설정되는대로 바로 반영되어 다시 읽어온다

1. 조회가능한 정보
   - 충전기 id
   - 충전소명
   - 제조사명
   - 시리얼번호
   - 펌웨어버전
   - 상태
   - 마지막연결상태(last heartbeat 수신일시)
   - 오늘 충전량(kWh)
   - 액션"
## 충전기 검색	
- 충전기 목록에서 아래의 필드정보를 이용하여 충전기 목록을 조회할 수 있다
- 사용자가 입력한 키워드와 일부가 동일하더라도 검색결과에 포함되어야 한다
- 충전기 정보조회 결과는 충전기 목록 조회시와 동일하다

1. 검색에 사용되는 컬럼정보
   - 충전기 id
   - 충전소명
   - 제조사명
   - 시리얼번호
   - 펌웨어버전"
## 충전기 삭제
- 시스템 관리자는 충전기 목록에서 특정 충전기를 정보를 삭제할 수 있어야 한다
## 충전기의 heartbeat 수신일시 저장	
- 충전기가 주기적으로 전솧아는 heartbeat를 수신하여 수신된 일시를 저장하고 있어야 한다
## 충전기의 상태 모니터링	
- 충전기의 상태를 아래와 같이 정의하고 모니터링할 수 있다

1) 충전기 상태
   - 미등록 : 아직 등록되기전 상태(provisioning상태에 있거나 알 수 없는 충전기)
   - 대기 : 정상등록되어 대기중인 상태
   - 충전중 : 현재 충전중
   - 장애 : 장애 발생한 상태
   - 조치중 : 기사가 현장에 출동한 상태
   - 통신장애 : 충전기와 통신이 안되는 상태
   - 상태 미확인 : 알 수 없는 상태(등록은 됐으나 위의 상태이외의 미확인 상태)"
## offline 상태의 충전기 이력 저장	
- 충전기의 상태 모니터링중 충전기 상태가 아래의 경우에 해당하는 것은 offline정보를 별도로 저장하고 있어야 한다

1) offline로 간주하는 충전기 상태
   - 장애
   - 조치중
   - 통신장애
   - 상태 미확
2) 저장할 정보
   - offline으로 확정한 일시
   - 파트너 id
   - 충전소 id
   - 충전기 id
   - 충전기 상태"
## offline 상태의 충전기 목록 조회	
- offline 상태에 있는 충전기들의 목록을 조회할 수 있다

1) 조회기준
   - 기간 : 조회시간일/조회종료일
   - 조회기간이 설정되지 않은 경우 : 현재  offline상태의 충전기 목록을 보여줌
   - 조회기간이 설정된 경우 : 해당 기간에 1번이라도 offline 상태에 있었던 충전기 목록 보여줌
"
# 사용자 관리	
## 사용자 목록 조회	
- 회원가입한 사용자들의 목록을 조회할 수 있다. 한 화면당 기본 20개의 정보를 확인할 수 있다
## 사용자 검색	
- 사용자가 입력한 검색어와 일부 매칭되는 사용자 정보를 검색할 수 있다.

1) 검색 컬럼정보
   - 사용자 id
   - 사용자 이름
   - 사용자 email
   - 전화번
   - 역할 : cs / partner / customer
   - 상태 : active / pending / inactive"
## 사용자 정보 변경	
- 사용자 정보중 아래의 항목은 설정변경할 수 있다

1. 기본정보
   - 성
   - 이름
   - 이메일
   - 전화번호
   - 언어
   - 상태"
## 파트너 회원가입 승인	
- 파트너의 회원가입시 승인대기상태를 승인상태로 변경할 수 있다
- 승인이 되면 사용자의 상태값을 'activate'로 변경함"
## 사용자 결제카드 정보	
- 사용자가 등록한 카드정보를 출력할 수 있다
# 충전카드 관리	
## 충전카드 등록	
- 충전카드를 등록할 수 있다
## 충전카드 정보 변경	
- 충전카드의 내용중 아래 항목은 변경할 수 있다

1) 변경가능한 충전카드 항목
   - 타입
      + local
      + ISO 14443
      + ISO 15693
      + KeyCode
      + eMAID
      + Central
      + MacAddress
      + NoAuthorization
   - 상태
      + Accepted
      + Blocked
      + Expired
      + Invalid "
## 충전카드 삭제	
- 지정한 충전카드를 삭제할 수 있다
## 충전카드 일괄등록	
- csv파일을 통하여 충전카드를 일괄 등록시킬 수 있다
- 이 기능을 통해 등록된 카드는 기존 카드정보에 추가된다(append)"
## 충전카드 예제 일괄등록파일 다운로드	
- 충전카드 일괄등록시 사용할 csv파일의 예제파일을 다운로드할 수 있다.
- 관리자는 이 파일의 포맷(컬럼명)에 맞게 csv파일을 구성해야 한다"
## 충전카드 검색	
- 충전카드 목록조회 화면에서 아래 필드를 이용하여 검색할 수 있다. 
- 사용자가 입력한 검색어와 일부분 매치되어도 검색결과에 포함시킨다

1) 검색가능 필드
   - RF ID 
   - 사용자 id"
