# "파트너(PARTNER)" 포탈에서 구현되어야 할 사항들 점검
## 대시보드	
### 대시보드 출력데이터	
- 파트너 소속의 충전현황 데이터를 요청기간에 대하여 보여준다.
1) 파트너 소속의 충전현황 데이터
   - 내 충전기 수 / 소속 전체 충전기
   - 온라인 충전기 / 현재 접속중
   - 당월 충전량 / 이번 달 충전량(kWh)
   - 당월 충전금액 / 이번달 충전금액"
## 내 충전소	
### 충전소 목록 조회
- 파트너에게 소속된 충전소 목록을 조회할 수 있다

1) 조회정보
   - 충전소명
   - 주소
   - 충전기 수
   - 충전단가(kWh당 단가)"
		충전소(site)별 충전단가 설정	파트너에게 소속된 충전소 단위로 충전단가(kWh당)를 설정할 수 있다
		충전소별 충전단가 적용	변경된 충전단가는 변경이후에 시작된 충전건부터 적용된다
		충전소별 충전단가 변경이력 조회	충전소별로 변경된 충전단가는 변경완료시마다 저장하여 변경이력을 확인할 수 있어야 한다
## 내 충전기	
- 충전기 목록 조회	"파트너에게 소속된 충전기 목록을 조회할 수 있다.

1) 조회정보
   - 충전기id
   - 충전소명
   - 상태
   - 마지막 연결"
## 충전 통계	
### 충전통계 트랜드	
- 파트너에 소속된 충전기들에 대한 충전 통계에 대한 트렌드를 확인할 수 있다

1) 충전통계항목
   - 충전건수
   - 충전량
   - 충전금액"
## 정산 내역	
### 청산 내역 조회	
- 파트너에 소속된 충전기들에 대한 정산내역을 확인할 수 있다

1) 정산내역
   - 정산일
   - 충전소명
   - 충전량(kWh)
   - 충전금액
   - 정산금액
   - 상태(정산상태)
2) 조회기간
   - 시작일/종료일
   - preset : 1주일(7일) / 1달 / 3달 / 6개월 / 1년"
		정산 내역 엑셀 다운로드	조회한 내용을 excel로 다운로드 할 수 있다
## 계좌정보	
### 계좌정보 등록	
- 정산금액이 입금되는 계좌정보를 등록할 수 있다

1) 등록할 계좌정보
   - 은행명
   - 계좌번호
   - 예금주"
		계좌정보 변경	등록된 계좌정보를 변경할 수 있다
		계좌정보 유효성 검증	

# 사용자(USER) 포털에서 구현되어야 할 사항들 점검	
## 대시보드	
### 대시보드 출력데이터	
- 고객이 사용한 충전정보를 보여준다
- 조회기간에 대한 정보를 보여준다

1) 조회기간
   - preset : 1일 / 1주일 / 1달 / 3달 / 6달 / 1년 / 전체
2) 출력정보
   - 총 충전건수 / 누적 충전건
   - 총 충전량 / 누적 충전량
   - 총 충전금액 / 누적 충전금액
   - 등록RFID 카드수(조회기간과 상관없음)
3) 최근 충전이력
   - 충전일시
   - 충전소명
   - 충전량(kWh)
   - 충전금액
   - 상태"
## 충전이력	
### 충전이력 조회	
- 조회하는 기간에 따라 고객이 충전한 충전이력을 조회한다

1) 조회기간
   - preset : 1일 / 1주일 / 1달 / 3달 / 6달 / 1년 / 전체
   - manual : 시작일 / 종료일
2) 출력항목
   - 충전일시
   - 충전소명
   - 충전량(kWh)
   - 충전금액
   - 상태"
## 결제카드	
### 결제카드 등록	
- 고객이 충전시 충전금액을 결제할 카드 또는 계좌인출을 위한 계좌정보를 등록한다
- 다수의 결재 수단이 등록되면 결제에 사용되는 카드 또는 계좌를 선택해서 default 결제수단을 정할 수 있다.

1) 카드등록정보
   - 카드번호
   - 유효기간
   - 카드 소유자
2) 계좌등록정보
   - 은행명
   - 계좌번호
   - 계좌 소유자 

### 결제카드 목록 조회	
- 고객이 등록한 카드목록을 조회할 수 있다

1) 카드정보
   - 카드사
   - 카드번호
   - 등록일"
### 결제카드 삭제
- 결제카드는 삭제할 수 있다
- 유효한 카드가 1개만 남아있는 경우에는 삭제할 수 없다"
