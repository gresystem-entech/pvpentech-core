# 결제금액(PAYMENT)
## 사용자 결제금액	
### 사용자의 결제금액 계산	
- 사용자의 결제금액 대상은 무엇인가?

1) 사용전력량 구매: 예정 사용전략량 * 충전단가(kWh) => 예정량에 이르면 충전기에게 충전종료 요청해야 함
2) 일정 비용만큼 충전시 선 결제금액 지정 : 예정 결제금액만큼 충전이 되면 충전기에게 충전종료 요청해야 함
	
## 사용자의 환불요청 처리	
### 사용자의 충전건에 대한 환불처리	
- 사용자의 충전 비용을 선 결제했으므로 충전 종료 시 충전량과 비교해서 금액이 남으면 환불처리해 주어야 한다

1) 환불요청하는 경우별 환불금액 결정방법
   - 사용자 결제금액 - 정산대상금액 = 환불금액"
### 환불이력 조회	
- 환불요청이 들어온 건들에 대한 환불이 처리상태를 확인할 수 있어야 한다

1) 환불이력에 대해서 출력이 필요한 정보
   - 파트너
   - 충전소
   - 충전기
   - 사용자
   - 충전일시
   - 결제금액
   - 충전금액
   - 환불금액
   - 환불처리상태"
## 파트너 정산금액	
### 파트너의 정산금액 계산	
- 파트너 소속의 충전기들에서 충전 transaction들의 아래의 금액합계를 파트너에게 정산할 금액으로 한다

1) 충전Transaction의 정산금액 
   - (사용자의 결제금액 - 사용자의 환불금액) * 리베이트율(마진율)"

# 정산(SETTLEMENT)
## 정산 처리	
### 일/주/월 단위 정산	
- 파트너별로 일/주/월 단위로 정산할 수 있다
- 정산대상 transaction은, 정산관련상태나 환불관련상태가 아닌 상태의 transation들이다

1) 일 : 
   - 이전의 정산대상 충전 transaction들에 대해서 정산을 진행한다(시간은 정해야 함)
2) 주
   - 정산할 요일을 정해야 한다
3) 월
   - 정산할 요일을 정해야 한다(2/29, 4/31과 같이 존재하지 않는 날자가 있는 경우에는 그 달의 마지막 날에 정산을 수행한다)"
