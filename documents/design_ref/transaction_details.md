# 정산 관리	
## 정산 현황 전체 조회	
- 전체 충전소에 대한 정산 현황목록을 조회할 수 있다
- 정산기간이란, 정산일이 포함되는 기간을 말한다

1. 정산 현황 유형
   - 전체 : 모든 충전소에 대한 정산유형
   - 파트너별 : 특정 파트너에 대한 충전소 정산유형
   - 충전소별 : 특정 충전소에 대한 정산 유형
2. 정산 현황 유형별 출력 정보
   1) 전체
      + 파트너명
      + 정산유형
      + 기간: 정산시작 ~ 정산종료
      + 총 충전량(kWh)
      + 정산금액
      + 상태
      + 정산일
   2) 파트너별 : 검색조건(파트너 && 정산시작일 && 정산종료일)
      + 파트너 명
      + 정산유형
      + 기간 : 시작일/종료일
      + 총 충전량(kWh)
      + 정산금액
      + 상태
   3) 충전소별 : 검색조건(충전소 && 정산시작일 && 정산종료일)
      + 충전소명
      + 파트너명
      + 기간 : 시작일/종료일
      + 총 충전량(kWh)
      + 정산금액
      + 상태"
## 수동 정산	
- 파트너의 미정산된 충전이력에 대하여 현시점을 기준으로 정산처리함

1) 파트너별 검색후 > `수동 정산` 버튼 필요
   - 현재 파트너의 충전transaction중 결제가 완료되지 않은 것들에 대해서 정산(총 정산금액 계산 + 계좌이체)작업을 진행한다"
# 충전 이력	
## 충전 이력 조회	
- 조회기준에 따라 충전이력에 대하여 조회할 수 있다

1) 검색 기준
   - 파트너
   - 충전소
   - 충전기
2) 검색 기준간 연관관계
   - 파트너 조건이 설정된 경우에는, 해당 파트너의 충전소 목록만 표시
   - 충전소 조건이 설정된 경우에는, 해당 충전소내의 충전기 목록만 표시
3) 조회정보
   - 충전일시
   - 파트너
   - 충전소명
   - 충전기ID
   - 총 충전량(kWh)
   - 충전금액
   - 리베이트율
   - 충전단가(kWh)
   - 상태"
## 충전이력에 대한 결제처리 결과확인	
- 충전이력(충전 transaction)에 대해서 사용자의 결제에 대하여 처리된 결과를 확인할 수 있다

1) 충전에 대한 사용자의 결제처리결과 확인
   - 충전이력 정보중 상태값이 아래중 하나여야 한다
      + pending
      + paid
      + failed
      + cancelled
      + refunded"
## 사용자 결제정보 저장	
- 사용자가 모든 충전transaction 건에 대해서 결제한 정보를 저장하고 있어야 한다

1) 결제저장정보
   - 결제수단
   - 결제시간
   - 결제금액
   - 거래번호"
## 사용자 결제정보 조회	사용자가 충전건에 대해서 결제한 내용을 확인할 수 있어야 한다
## 충전 이력 조회시 조회범위 검색	
- 파트너나 충전소를 대상으로 검색을 수행하여 사용자가 입력한 검색어와 일부 매칭되는 파트너/충전소 목록을 보여준다.
- 출력된 목록중 사용자가 원하는 항목 선택시, 선택된 파트너와 충전소 정보로 조회조건이 설정된다"
